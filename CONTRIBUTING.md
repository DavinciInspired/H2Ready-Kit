# Contributing to H2Ready

Thank you for your interest in contributing to H2Ready.

## Ways to Contribute

- Propose improvements to the scoring logic or penalty rules
- Add new visualisations (e.g. portfolio analytics, uncertainty overlays)
- Implement data importers for ILI, CP and SCADA data
- Extend the documentation or add validation case studies
- Experiment with Level‑2 / Level‑3 ML models in separate branches

## Development Setup

1. Fork the repository.
2. Clone your fork and create a feature branch.
3. Run the stack locally using Docker:
   ```bash
   docker compose up --build
   ```
4. Make changes, add tests where appropriate, and open a Pull Request.

## Code of Conduct

Please keep discussions professional and constructive. Respect differing viewpoints,
especially regarding standards, safety margins and modelling approaches.
